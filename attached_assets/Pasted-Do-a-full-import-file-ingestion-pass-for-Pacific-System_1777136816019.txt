IMPORT SOURCES
━━━━━━━━━━━━━━━━━━━━

Support these import formats:

FORMAT A — Standard CSV:
Name,Number,Date,Time,Duration,Status,Notes

FORMAT B — MicroSIP CSV Export:
Type,Name,Number,Time,Duration,Info

MicroSIP details:
- Type can be: out, in, miss, else
- Time is Unix timestamp seconds
- Duration is seconds
- Info is call status/detail

Map MicroSIP Type:
- out = Outgoing
- in = Answered
- miss = Missed
- else = Other / Answered Elsewhere

FORMAT C — MicroSIP INI call lines:
[Calls]
0=number;name;type;unix_time;duration_seconds;info
1=number;name;type;unix_time;duration_seconds;info

INI type mapping:
- 0 = Outgoing
- 1 = Answered
- 2 = Missed
- 3 = Other / Answered Elsewhere

Ignore:
- [Calls]
- [Settings]
- callsLastKey
- blank lines
- unrelated settings

FORMAT D — Callcentric-style CSV / copied report:
Try to detect common columns:
Name, Number, Date, Time, Duration, Info, Status, Direction
or similar.

If the parser cannot detect it, show a clear error explaining the expected columns.

━━━━━━━━━━━━━━━━━━━━
DEDUPLICATION
━━━━━━━━━━━━━━━━━━━━

This is critical.

MicroSIP may export the full call history every time.
Callcentric may export overlapping logs.

The app must never duplicate the same call.

For every call, generate a stable dedupeKey.

Use:
- normalized phone number
- normalized caller name
- startedAt/date+time
- durationSeconds
- normalized status/type
- notes/info

Example:
`${number}|${name}|${startedAt}|${durationSeconds}|${status}|${notes}`

Normalize:
- strip spaces, dashes, parentheses from phone numbers
- lowercase caller name
- trim all fields
- convert dates/times to consistent ISO if possible
- convert duration to seconds

On import:
- Load existing calls
- Generate dedupeKey for each imported row
- If dedupeKey exists, skip it
- If dedupeKey does not exist, add it

If same number/name/time/status exists but notes differ slightly:
Still treat as duplicate if timestamp + number + duration + status match.

Add import result:
“Safe Import Enabled — duplicate calls were skipped.”

Test case:
If I import the same full MicroSIP CSV twice:
First import: Imported X, skipped 0
Second import: Imported 0, skipped X
